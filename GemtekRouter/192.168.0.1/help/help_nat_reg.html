Configuración del redireccionamiento de puertos

Aunque la Device sólo realiza la traducción de direcciones en las comunicaciones iniciadas desde la LAN , para algunas aplicaciones, como los juegos en red, es necesario autorizar las comunicaciones desde la INTERNET a través de la (NAT estática). Edite con cuidado las reglas para la comunicación desde fuera de la red interna con el dispositivo de red de la LAN (NAT estática) y, si es necesario, consulte la documentación del juego. Se pueden registrar hasta 32 reglas.


Agregar/editar redireccionamiento de puertos

Permite agregar nueva información de redireccionamiento de puertos y modificar la existente.


Grupo

Puede dar un nombre (nombre de grupo) a cada NAT estática configurada y proporcionar un mismo nombre a un grupo de NAT estática y administrarlas juntas.
Al proporcionar nombres a los grupos, puede [Habilitar] o [Deshabilitar] cada uno por separado.
Para agregar una regla de NAT estática al grupo existente, seleccione el grupo en el cuadro desplegable y elija [Agregar].
Para crear un nuevo grupo, seleccione [Nuevo grupo] para la tabla y escriba el nombre del nuevo grupo en el cuadro de diálogo.
El nombre de grupo puede tener hasta 16 caracteres alfanuméricos. Los valores predeterminados son nuevo y en blanco.


Memo

Si se agrega un nombre nuevo en blanco, automáticamente se proporciona un nombre con el formato [Grupo]+[Número]. No se puede especificar un nombre en blanco durante la edición.


Dirección IP de la INTERNET

Permite especificar la dirección de destino de la NAT estática del paquete de comunicación. La Device aplica la NAT estática en las comunicaciones cuyo destino es esta dirección.

Puede seleccionar las opciones siguientes.
[Dirección IP de la INTERNET de la Device]
[Configuración manual]

Cuando se obtiene una dirección IP de la INTERNET de un servidor PPPoE, se pueden especificar las direcciones IP de la INTERNET de cada destino PPPoE.
Cuando se usa la configuración manual, se debe especificar una dirección IP en el cuadro de diálogo.
De forma predeterminada, el campo de dirección IP manual está en blanco.


Protocolo

Permite seleccionar el protocolo que se analizará.
Se puede seleccionar cualquiera de las opciones siguientes.

  • Todo
    Se analizarán todos los protocolos.
  • ICMP
    Se analizarán los paquetes ICMP (protocolo 1).
  • Manual
    Seleccione el número de protocolo (1-255) que se analizará.
    De forma predeterminada, este campo está en blanco. Si se selecciona la opción manual, escriba un número. Si no se selecciona la opción manual, no se tiene en cuenta el número introducido.
  • TCP/UDP
    Se analizan los protocolos TCP (protocolo 6) o UDP (protocolo 17).
    Con TCP/UDP, también puede usar un número de puerto. Si no se selecciona TCP/UDP, no se tiene en cuenta el número de puerto.


El valor predeterminado es TCP/UDP.


TCP/UDP

Cuando se seleccionan los protocolos TCP o UDP, también se debe seleccionar el número de puerto de destino.

  • Puerto TCP arbitrario
    Traduce el número de puerto TCP especificado.
    Los números de puerto válidos están comprendidos entre 1 y 65535.
  • Puerto UDP arbitrario
    Traduce el puerto UDP especificado.
    Los números de puerto válidos están comprendidos entre 1 y 65535.
  • HTTP (Puerto TCP: 80)
    Traduce las comunicaciones con el servidor HTTP. Esto equivale a especificar el puerto TCP 80. (No es necesario especificar el número de puerto).
  • FTP (Puerto TCP: 20-21)
    Traduce las comunicaciones con el servidor FTP. Esto equivale a especificar los puertos TCP 20-21. (No es necesario especificar el número de puerto).
  • HTTPS (Puerto TCP: 443)
    Traduce las comunicaciones con el servidor HTTPS. Esto equivale a especificar el puerto TCP 443. (No es necesario especificar el número de puerto).
  • TELNET (Puerto TCP: 23)
    Traduce las comunicaciones con el servidor TELNET. Esto equivale a especificar el puerto TCP 23. (No es necesario especificar el número de puerto).
  • SSH (Puerto TCP: 22)
    Traduce las comunicaciones con el servidor SSH. Esto equivale a especificar el puerto TCP 22. (No es necesario especificar el número de puerto).
  • SMTP (Puerto TCP: 25)
    Traduce las comunicaciones con el servidor SMTP. Esto equivale a especificar el puerto TCP 25. (No es necesario especificar el número de puerto).
  • POP3 (Puerto TCP: 110)
    Traduce las comunicaciones con el servidor POP3. Esto equivale a especificar el puerto TCP 110. (No es necesario especificar el número de puerto).
  • NTP (Puerto UDP: 123)
    Traduce las comunicaciones con el servidor NTP. Esto equivale a especificar el puerto UDP 123. (No es necesario especificar el número de puerto).
  • NTP (Puerto TCP: 123)
    Traduce las comunicaciones con el servidor NTP. Esto equivale a especificar el puerto TCP 123. (No es necesario especificar el número de puerto).
  • IMAP (Puerto TCP: 143)
    Traduce las comunicaciones con el servidor IMAP. Esto equivale a especificar el puerto TCP 143. (No es necesario especificar el número de puerto).



Dirección IP de la LAN

Escriba la dirección IP del dispositivo de la LAN en el que se aplica la NAT estática. Escriba la dirección IP del servidor que se va a usar para el juego en red del PC, etc. De forma predeterminada, este valor está en blanco, pero se puede especificar en caso necesario.
Aquí se configura la dirección IP de destino de los paquetes convertidos a la dirección IP de la LAN durante el redireccionamiento de puertos.
Si una dirección IP se distribuye dinámicamente desde un servidor DHCP y la dirección IP del equipo ha cambiado porque la concesión DHCP ha vencido, es posible que sea necesario volver a configurar el redireccionamiento de puertos. En este caso, se recomienda configurar DHCP para que proporcione una dirección IP fija al dispositivo de red de la LAN usado en el redireccionamiento de puertos.


Nota:

Si la dirección IP de la LAN está configurada en otro segmento de red, los paquetes no se podrán transferir a los dispositivos de la LAN.


Puerto de la LAN

Cuando se especifica TCP/UDP como protocolo de redireccionamiento y se indica un solo número de　puerto, se puede cambiar el número de puerto de la LAN.
Este valor está en blanco de forma predeterminada, y los números de puerto de los paquetes de comunicación no cambian.
Puede establecer números de puerto comprendidos entre 1 y 65535. No es posible especificar varios números de puerto a la vez.

Información de registro del redireccionamiento de puertos

Puede especificar [Habilitar]/[Deshabilitar] para cada grupo.

Botón [Deshabilitar]

Al hacer clic en el botón [Deshabilitar], el elemento de redireccionamiento de puertos no se registra.
El botón [Deshabilitar] se muestra con el estado [Habilitar].


Botón [Habilitar]

Al hacer clic en [Habilitar] se registra un elemento de redireccionamiento de puertos.



Botón [Editar]

Haga clic en [Editar] para cambiar una entrada. La línea se volverá roja. Finalizada la edición, haga clic en [Aplicar] para mantener los cambios o en [Cancelar] para descartarlos.


Botón [Eliminar]

Haga clic en [Eliminar] para eliminar una línea.
