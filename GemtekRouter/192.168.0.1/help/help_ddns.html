Configuración de DNS dinámico

Configuración de DNS dinámico.
Antes de configurar estas opciones, es necesario registrarse con un proveedor de servicio DNS dinámico.


Servicio DNS dinámico

Seleccione un proveedor de servicio DNS dinámico.
Puede seleccionar "DynDNS" o "TZO".

  • DynDNS
  • TZO


Los siguientes valores son diferentes según el proveedor del servicio DNS dinámico.


DynDNS

Nombre de usuario, contraseña, nombre de host

Escriba la información recibida de DynDNS.


Nombre de usuario

Se pueden introducir 64 caracteres como máximo.


Contraseña

Se pueden introducir 64 caracteres como máximo.


Nombre de host

Se pueden usar caracteres alfanuméricos, puntos "." y guiones "-". La longitud máxima son 255 caracteres. La cadena dividida por , puntos "." no debe comenzar ni terminar con un guión.


Periodo de actualización de direcciones IP

Periodo durante el cual la Device notifica al proveedor del servicio DNS dinámico una dirección IP.
El intervalo de valores válidos es de 0 a 35 días.
El valor predeterminado es 7 días. Cuando se establece en 0 días, no se realizará ninguna actualización periódica de direcciones.


TZO

Dirección de correo electrónico, contraseña de TZO, nombre de dominio

Escriba la información con la que se ha registrado en TZO.


Dirección de correo electrónico

Se pueden introducir 64 caracteres como máximo.


TZO Key

Se pueden introducir 64 caracteres como máximo.


Nombre de dominio

El nombre de dominio puede tener una longitud de hasta 255 caracteres alfanuméricos, y se permite usar guiones "-" y puntos ".", pero el nombre no puede comenzar ni acabar con ninguno de esos caracteres.


Periodo de actualización de direcciones IP

Periodo durante el cual la Device notifica al proveedor del servicio DNS dinámico una dirección IP.
El intervalo de valores válidos es de 0 a 99 días.
El valor predeterminado es 7 días. Cuando se establece en 0 días, no se realizará ninguna actualización periódica de direcciones.


Nota:

Es necesario notificar periódicamente la dirección IP al proveedor del servicio DNS dinámico. Cuando el periodo de actualización es demasiado largo, la información del dominio se puede eliminar del servicio DNS dinámico.


Cuando se actualiza la dirección IP de la INTERNET (por ejemplo, conectando de nuevo el PPPoE), la Device notifica una dirección IP al proveedor del servicio DNS dinámico, con independencia del periodo de actualización de direcciones IP. En ese momento, se calcula el siguiente periodo de actualización.


No se puede realizar la notificación de direcciones IP en las siguientes circunstancias.

  • La dirección IP de la INTERNET es una dirección IP privada.
  • La dirección IP de la INTERNET se obtiene mediante una IP sin numerar.
  • La traducción de direcciones de red está deshabilitada.


Incluso si la configuración es correcta, "Error de actualización" podría aparecer por los motivos que se enumeran a continuación.

  • Cuando el proveedor del servicio de DNS dinámico rechaza la configuración. (Es posible que el proveedor del servicio de DNS dinámico haya rechazado el registro continuo.)
  • Cuando no se puede detectar el nombre de dominio registrado en el proveedor del servicio de DNS dinámico. (La notificación a todos los servidores de DNS puede durar unos cuantos días. El nombre de dominio no se detecta inmediatamente después de registrarse en el servicio de DNS dinámico.)


Información actual de DNS dinámico

Dirección IP de la INTERNET

Dirección IP del Puerto INTERNET de la Device actual.
La Device notifica esta dirección al proveedor del servicio DNS dinámico.


Nombre de dominio

Nombre de dominio asignado por el proveedor del servicio DNS dinámico.
Con este nombre de dominio es posible tener acceso a la Device desde Internet.


Estado

El estado de la conexión del servicio DNS dinámico


Botón [Actualizar]

Al hacer clic en este botón, se actualizará la información del DNS dinámico mostrada.
