Configuración de filtro de IP

Limita el tipo de paquete que se permite pasar entre la INTERNET y la LAN.
El número máximo de reglas es 32.
Si el paquete cumple una de las condiciones de control (consulte a continuación) antes de que se enrute, se realizará la acción especificada. Si se cumplen varias condiciones (consulte a continuación), la acción adecuada se realizará una vez que el paquete cumpla la condición.


Resultado del registro

Al activar esta casilla se registrará la información de filtro de IP en un registro. Si se acepta el funcionamiento, se deshabilita el resultado del registro.
El valor predeterminado es Deshabilitado.


Agregar/editar filtro basado en dirección IP

Este área sirve para agregar o editar líneas.


Funcionamiento

Seleccione la acción que se realizará en los paquetes que cumplen los criterios de filtro.
Omitido
Detiene el paquete y no se enruta.
Rechazado
Devuelve el paquete rechazado al
punto de origen.
Aceptado
Se permite el paso del paquete.
El valor predeterminado es [Omitido].


Dirección

Especifique la dirección de la secuencia de paquetes.
LAN→INTERNET
Se filtrarán los paquetes procedentes de la LAN.
INTERNET→LAN
Se filtrarán los paquetes procedentes de la INTERNET.
El valor predeterminado es "INTERNET→LAN".
Con la dirección se pueden especificar las condiciones siguientes. En este caso, se filtran los paquetes que cumplen con todas las condiciones.


Dirección IP

Se puede designar una dirección IP de origen y destino para el filtro.
De forma predeterminada, este campo está en blanco.
Para filtrar todas las direcciones IP, deje este cuadro en blanco.
En lugar de la dirección IP, se puede especificar la dirección de red.
En este caso, el valor de máscara detrás de "/" seguido de la dirección de red especificará la máscara de subred.
El valor de máscara está comprendido entre 1 y 32.


Ejemplo

Especifique 192.168.1.0 como dirección y 255.255.255.0 como la máscara de subred.

192.168.1.0/24


Protocolo

Permite seleccionar el protocolo que se analizará.
Se puede seleccionar cualquiera de las opciones siguientes.

  • Todo
    Se analizarán todos los protocolos.
  • ICMP
    Se analizarán los paquetes ICMP (protocolo 1).
  • Manual
    Seleccione el número de protocolo (1-255) que se analizará.
    De forma predeterminada, este campo está en blanco. Si se selecciona la opción manual, escriba un número. Si no se selecciona la opción manual, no se tiene en cuenta el número introducido.
  • TCP/UDP
    Se analizan los protocolos TCP (protocolo 6) o UDP (protocolo 17).
    Con TCP/UDP, también puede usar un número de puerto. Si no se selecciona TCP/UDP, no se tiene en cuenta el número de puerto.


El valor predeterminado es TCP/UDP.


TCP/UDP

Cuando se seleccionan los protocolos TCP o UDP, también se debe seleccionar el número de puerto de destino.

  • Puerto TCP arbitrario
    Traduce el número de puerto TCP especificado.
    Los números de puerto válidos están comprendidos entre 1 y 65535.
  • Puerto UDP arbitrario
    Traduce el puerto UDP especificado.
    Los números de puerto válidos están comprendidos entre 1 y 65535.
  • HTTP (Puerto TCP: 80)
    Traduce las comunicaciones con el servidor HTTP. Esto equivale a especificar el puerto TCP 80. (No es necesario especificar el número de puerto).
  • FTP (Puerto TCP: 20-21)
    Traduce las comunicaciones con el servidor FTP. Esto equivale a especificar los puertos TCP 20-21. (No es necesario especificar el número de puerto).
  • HTTPS (Puerto TCP: 443)
    Traduce las comunicaciones con el servidor HTTPS. Esto equivale a especificar el puerto TCP 443. (No es necesario especificar el número de puerto).
  • TELNET (Puerto TCP: 23)
    Traduce las comunicaciones con el servidor TELNET. Esto equivale a especificar el puerto TCP 23. (No es necesario especificar el número de puerto).
  • SSH (Puerto TCP: 22)
    Traduce las comunicaciones con el servidor SSH. Esto equivale a especificar el puerto TCP 22. (No es necesario especificar el número de puerto).
  • SMTP (Puerto TCP: 25)
    Traduce las comunicaciones con el servidor SMTP. Esto equivale a especificar el puerto TCP 25. (No es necesario especificar el número de puerto).
  • POP3 (Puerto TCP: 110)
    Traduce las comunicaciones con el servidor POP3. Esto equivale a especificar el puerto TCP 110. (No es necesario especificar el número de puerto).
  • NTP (Puerto UDP: 123)
    Traduce las comunicaciones con el servidor NTP. Esto equivale a especificar el puerto UDP 123. (No es necesario especificar el número de puerto).
  • NTP (Puerto TCP: 123)
    Traduce las comunicaciones con el servidor NTP. Esto equivale a especificar el puerto TCP 123. (No es necesario especificar el número de puerto).
  • IMAP (Puerto TCP: 143)
    Traduce las comunicaciones con el servidor IMAP. Esto equivale a especificar el puerto TCP 143. (No es necesario especificar el número de puerto).

Información del filtro de IP

Muestra el filtro basado en IP registrado. En cada línea se muestra también el número de paquetes filtrados.
Este número se borra cuando se cambia la configuración.


Botón [Editar]

Haga clic en [Editar] para cambiar una entrada. La línea se volverá roja. Finalizada la edición, haga clic en [Aplicar] para mantener los cambios o en [Cancelar] para descartarlos.


Botón [Eliminar]

Haga clic en [Eliminar] para eliminar una línea.

Botón de flecha abajo
